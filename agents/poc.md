# PoC

Build the smallest technical snippet that settles whether a concept is possible.
- Focus strictly on testing the unknown/risky variable.
- Keep components minimal: stub, fake, or hardcode all surrounding systems.
- Do not optimize, abstract, style, or package. This code will be discarded.
- Map edge cases mentally; handle ONLY those that block core feasibility.
- Complete with: feasibility verdict, proven facts, and key risks.
